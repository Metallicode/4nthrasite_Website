Beta access — v0.1.0

Welcome, beta tester.

Thank you for being here. Your feedback during this early phase is what shapes 4nthracite into the instrument it should be.

4nthracite is a generative music workstation built as a native desktop app — there is no web version and no internet connection required after installation. Select your platform below and follow the steps to get running.

Select your platform

Download v0.1.0 — Windows 10/11 (64-bit)

4nthracite_0.1.0_x64-setup.exe RECOMMENDED 4nthracite_0.1.0_x64_en-US.msi MSI

NSIS Installer (.exe) — Recommended

  1. Go to the GitHub Releases page and download 4nthracite_0.1.0_x64-setup.exe from the Assets section.
  2. Double-click the downloaded file to launch the installer.
  3. If Windows SmartScreen shows a warning — "Windows protected your PC":

    a. Click More info

    b. Click Run anyway

    This appears because the app is not yet code-signed. It is safe to proceed.

  4. Follow the installer prompts: Next → Install → Finish.
  5. Launch 4nthracite from the Start Menu or the desktop shortcut.

SmartScreen warning — expected

Windows will warn that the app is from an unknown publisher. This is normal for unsigned beta software. Click More info → Run anyway to proceed.

MSI installer — alternative method

MSI Package — Alternative

  1. Download 4nthracite_0.1.0_x64_en-US.msi from the Releases page.
  2. Double-click the .msi file.
  3. Accept the SmartScreen prompt as above (More info → Run anyway).
  4. Follow the installer wizard.
  5. Launch from the Start Menu.

WebView2 runtime

Most Windows 10/11 machines already have WebView2 installed (it ships with Edge). If the app fails to launch with a WebView2 error, download the Evergreen Bootstrapper from microsoft.com/edge/webview2.

Audio output

On Windows, 4nthracite supports selecting the audio output device from within the Mixer panel. Use the device picker to route audio to a specific interface (WASAPI / DirectSound).

Uninstall

Uninstall Settings → Apps → 4nthracite → Uninstall
Min. requirement Windows 10 64-bit with WebView2 runtime

Download v0.1.0 — macOS 11 Big Sur or later

4nthracite_0.1.0_aarch64.dmg APPLE SILICON 4nthracite_0.1.0_x64.dmg INTEL

Which file do I need?

Apple menu → About This Mac → check Chip or Processor.
M1 / M2 / M3 / M4 → aarch64.dmg  ·  Intel → x64.dmg

Installation

  1. Download the .dmg that matches your Mac (see above).
  2. Double-click the downloaded .dmg to mount it.
  3. Drag the 4nthracite icon into the Applications folder.
  4. Eject the disk image.
  5. Open Finder → Applications and double-click 4nthracite.

Gatekeeper warning — first launch

Because the app is not yet notarised, macOS will block it on first open: "4nthracite cannot be opened because it is from an unidentified developer."

Option A (recommended):

In Finder, right-click (Control-click) the 4nthracite app icon → Open → click Open in the dialog. You only need to do this once.

Option B (System Settings):

Try opening normally (it will be blocked) → System Settings → Privacy & Security → scroll to Security → click Open Anyway → authenticate.

Audio output — important

On macOS, in-app audio device selection is not available (WebKit/Safari limitation). To route audio to a specific output (e.g. an audio interface), set it in System Settings → Sound → Output before launching the app.

Uninstall

Uninstall Drag 4nthracite from Applications to the Trash
Min. requirement macOS 11 Big Sur or later

Ubuntu 20.04 not supported

4nthracite requires WebKit2GTK 4.1, which is not available on Ubuntu 20.04. Please upgrade to Ubuntu 22.04 LTS or use a newer distribution.

Download v0.1.0 — x86_64, Ubuntu 22.04+ or equivalent

4nthracite_0.1.0_amd64.AppImage RECOMMENDED 4nthracite_0.1.0_amd64.deb DEB

AppImage — Recommended (works on most distros)

  1. Download 4nthracite_0.1.0_amd64.AppImage from the Releases page.
  2. Open a terminal and make it executable: 
    chmod +x ~/Downloads/4nthracite_0.1.0_amd64.AppImage
  3. Run it: 
    ~/Downloads/4nthracite_0.1.0_amd64.AppImage
    Or double-click it in your file manager (if AppImage support is enabled).

FUSE required for AppImage

AppImages need FUSE to run. If you get a FUSE or libfuse error:

Ubuntu/Debian: sudo apt install libfuse2
Fedora: sudo dnf install fuse
Arch: sudo pacman -S fuse2

Optional — integrate into app launcher
  1. Move the AppImage to a permanent location: 
    mkdir -p ~/Applications
mv ~/Downloads/4nthracite_0.1.0_amd64.AppImage ~/Applications/
  2. Right-click the file in your file manager → Properties → Permissions → tick Allow executing as program.
  3. To add to your application menu, install AppImageLauncher, then double-click the AppImage — it will prompt to integrate.
.deb package — Ubuntu / Debian / Linux Mint
  1. Download 4nthracite_0.1.0_amd64.deb from the Releases page.
  2. Install via terminal: 
    sudo dpkg -i ~/Downloads/4nthracite_0.1.0_amd64.deb
    Or double-click the .deb in Files/Nautilus to open the Software Center.
  3. If installation fails with missing dependencies, run: 
    sudo apt --fix-broken install
    Or install the dependency manually: 
    sudo apt install libwebkit2gtk-4.1-0
  4. Launch from your application menu, or run: 
    4nthracite

Audio output

4nthracite outputs to the default PulseAudio or PipeWire device. If you get no sound, check that your audio server is running:

pulseaudio --check  or  systemctl --user status pipewire

AppImage Delete the .AppImage file — no system files are created

Sessions & data

On first launch, 4nthracite will prompt you to create or open a session. Session files are stored at:

Linux / macOS ~/Documents/Anthracite/{session-name}/
Windows C:\Users\{YourName}\Documents\Anthracite\{session-name}\

Each session folder contains:

{name}.anthc Full session state (JSON)
{name}.anthc.autosave Auto-saved every 3 minutes
recordings/ WAV exports from the REC button
logs/ Timestamped log files — one per app run

Report a bug

If you hit a crash or unexpected behaviour, please include the most recent .log file from your session's logs/ folder.

Useful info to include:

  • Your OS and version (e.g. Windows 11, macOS 14.4, Ubuntu 22.04)
  • The most recent .log file from logs/
  • Steps to reproduce the issue
  • What you expected vs. what happened
